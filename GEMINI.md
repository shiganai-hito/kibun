# プロジェクト・コンテキスト: kibun

## 1. 開発環境
- **IDE:** Android Studio (最新安定版)
- **言語:** Kotlin (2.0.21)
- **UIフレームワーク:** Jetpack Compose / Viewシステム (両方。新規画面はComposeを優先)
- **アーキテクチャ:** MVVM (data/ui の構成に基づく)
- **依存関係管理:** Version Catalog (libs.versions.toml)

## 2. 実装ルール
- **非同期処理:** Kotlin Coroutines / Flow を使用。
- **DI (依存性の注入):** なし (現時点)
- **ネットワーク:** なし (現時点)
- **画像読み込み:** なし (現時点)
- **命名規則:**
    - クラス名は PascalCase。
    - 関数・変数は camelCase。
    - Composable関数は PascalCase。

## 3. Geminiへの要望（プロンプト・ガイドライン）
あなたは熟練のAndroidエンジニアとして振る舞ってください。

### 言語設定:
- 原則として**日本語**で回答してください。

### 回答の優先事項:
1. **安全性:** Null安全、例外処理、メモリリークの防止を最優先する。
2. **モダン:** 古いAPI（非推奨）を避け、最新のAndroid SDK/ライブラリを提案する。
3. **簡潔:** コードは読みやすく、ボイラープレート（定型文）を最小限にする。
4. **テスト:** 必要に応じて、ユニットテストやUIテストの書き方も提示する。

### 禁止事項:
- `findViewById` の使用（Viewの場合はViewBinding、基本はComposeを優先）。
- 命令的なUI更新（可能な限り宣言的UIで記述）。
- 複雑すぎるネスト。

## 4. 特記事項
- **オフラインファースト:** Room (KibunDatabase) を使用し、ローカルデータの整合性を重視します。
- **ダークモード:** ダークモード対応を必須とします。
- **以前の変更:** 以前のセッションでカレンダー機能が削除されています。
